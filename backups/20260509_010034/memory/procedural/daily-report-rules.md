# 每日简报（早报）规则 - 2026-03-12更新版

## ⏰ 发送时间
**每天早上 7:50**（避开8:00其他任务）

---

## 📋 简报固定结构

### 1. 🔴 私人待办（优先级最高）
- 放在第一条
- 个人生活事务
- 最高优先级处理
- 戒烟打卡（第N天）

### 2. 🟡 工作待办（按优先级排序）
排序规则：
1. **🔴P0** - 最紧急重要
2. **🟡P1** - 重要不紧急
3. **🟢P2** - 一般事项

### 3. 🟢 48小时提醒（提前预警）
包含内容：
- 未来48小时内的会议
- 客户拜访安排
- 截止日期提醒
- cron jobs中即将触发的任务

---

## 🎯 核心战役（当前真实工作）

### 战役一：推广AI老师傅
- 客户拜访、产品演示
- 商务洽谈、合同签署
- 市场拓展、品牌建设

### 战役二：产品升级迭代
- 从产品经理角度优化功能
- 用户体验改进
- 技术架构升级

### 战役三：融资准备（未来目标）
- 商业模式梳理
- 财务数据准备
- 投资人对接

---

## 🔄 任务跟踪规则

### 任务未完成处理
1. **用户告知未完成** → 自动顺延到下一天早报
2. **用户告知截止日期** → 按截止日期设置提醒

### 截止日期提醒策略
| 时间距离 | 提醒频率 |
|----------|----------|
| 距离较长 | 提前3天第一次提醒 |
| 最后3天 | **每天提醒** |
| 最后1天 | 重点标注，多次确认 |

**直到用户明确告知"已完成"为止**
