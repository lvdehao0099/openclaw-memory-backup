## 第四章：执行流程

### 4.1 标准流程

1. **使用 wechat-article-search skill 搜索关键词**
   - 每个关键词取前3条
   - 去重处理

2. **调用脚本写入飞书 Bitable**
   - 脚本路径：`/root/.openclaw/workspace/scripts/hotspot_batch_insert.py`
   - 传入 app_token 和 table_id

3. **验证写入成功**
   - 检查返回 `code == 0`
   - 如脚本失败，手动调用飞书 API 写入

### 4.2 数据保留策略

- **保留期：** 14天
- **自动清理：** 每天00:00清理过期记录
- **备份：** 重要热点手动标记为"长期保留"

## 第六章：选题工作流

### 6.1 每日早上（8:30）

1. 查看昨日20:00采集的微信热榜前10
2. 筛选高/中相关度热点
3. 结合AI老师傅业务生成选题
4. 将选中热点状态改为"已选题"

### 6.2 选题优先级算法

| 维度 | 权重 | 说明 |
|------|------|------|
| 热度排名 | 30% | 排名越靠前越好 |
| 相关度 | 40% | 高相关>中相关>低相关 |
| 时效性 | 20% | 3天内>7天内>14天内 |
| 可操作性 | 10% | 容易切入>需要包装 |

**推荐公式：**
```
选题价值 = 热度排名×0.3 + 相关度×0.4 + 时效性×0.2 + 可操作性×0.1
```

## 第七章：公众号文章创作流程

### 7.1 工作流程（必须严格执行）

| 阶段 | 时间 | 内容 |
|------|------|------|
| 准备 | 30min | 读取策略文档、参考资料 |
| 团队讨论 | 1-2h | 至少4轮讨论（选题/结构/表达/结尾）|
| 撰写初稿 | 2-3h | 文案虾主笔 |
| 内部审阅 | 30min | 老师傅虾审核事实准确性 |
| 路人审阅 | 1h | 至少2个路人甲审阅 |
| 修改完善 | 1-2h | 整合意见修改 |
| 定稿交付 | 30min | 创建飞书文档 |

### 7.2 常见错误（血泪教训）

| 错误 | 表现 | 后果 | 预防 |
|------|------|------|------|
| 主题偏离 | 凭记忆瞎编 | 完全重写 | 动笔前抄写策略要求 |
| 缺少路人审阅 | 团队内部自嗨 | 发布后被挑错 | 必须找2-3个路人甲 |
| 数据造假 | 编具体数字 | 信任崩塌 | 无来源用模糊表述 |
| 贬低他人 | 把人工经验说的一无是处 | 得罪目标受众 | 强调"辅助"不是"替代" |

### 7.3 质量检查清单

**内容质量：**
- [ ] 主题严格符合策略要求
- [ ] 无编造数据
- [ ] 无贬低竞品/他人
- [ ] 语气温和、专业

**流程完整：**
- [ ] 团队讨论记录完整
- [ ] 路人审阅记录完整
- [ ] 修改执行记录完整
- [ ] 文档在正确文件夹
