## 四、执行流程

### 4.1 新增待办流程

1. **用户提出待办**
   - 立即询问表格信息（如果未知）

2. **解析相对时间词**
   - 提取"明天/后天/下周X/周X"等相对时间词
   - 计算绝对日期，存储到「截止日期」字段
   - **剥离相对时间词，只保留核心内容存入「事项名称」**

3. **写入本地日志**
   - 路径：`memory/YYYY-MM-DD.md`

4. **写入飞书待办追踪表**
   - 使用飞书 API 写入
   - 日期字段使用毫秒时间戳

5. **确认双写成功**
   - 检查 API 返回 `code == 0`

### 4.2 更新待办流程

1. 查询飞书待办表获取 record_id
2. 更新状态字段
3. 同步更新本地日志

## 五、相对日期处理规则（重要！）

### 5.1 录入时转换（关键！）

**步骤1：提取相对时间词，计算绝对日期**

| 用户输入 | 计算逻辑 | 截止日期值 |
|---------|---------|----------|
| "明天下午" | 今天+1天 | 绝对日期时间戳 |
| "后天上午" | 今天+2天 | 绝对日期时间戳 |
| "下周三" | 下周三 | 绝对日期时间戳 |
| "周五" | 最近一个周五 | 绝对日期时间戳 |

**步骤2：剥离相对时间词，只保留核心内容**

| 用户输入 | 事项名称 | 截止日期 |
|---------|---------|---------|
| "明天要跟进大连永光" | "跟进大连永光" | 明天的时间戳 |
| "后天提醒我给客户打电话" | "给客户打电话" | 后天的时间戳 |
| "下周三跟进合同签署" | "跟进合同签署" | 下周三的时间戳 |

**剥离的相对时间词列表：**
- 明天、后天、大后天
- 下周一、下周二、...、下周日
- 周一、周二、...、周日
- 这周X、下周X
- 几号、X号

### 5.2 早报生成时判断

**使用「截止日期」字段判断时间，不依赖事项名称：**

| 截止日期 | 显示格式 |
|---------|---------|
| = 今天 | 显示在「工作待办」区块 |
| = 明天 | 显示在「未来提醒」区块，标注"明天" |
| = 后天 | 显示在「未来提醒」区块，标注"后天" |
| 其他 | 显示在「未来提醒」区块，标注"M月D日" |

## 六、提醒方式（重要！）

| 方式 | 是否允许 | 说明 |
|------|:--------:|------|
| ❌ 创建定时提醒 | 禁止 | 用户不需要到时间提醒 |
| ✅ 在早报中提醒 | 允许 | 所有日程类待办都在当天早报里提醒 |

**早报流程：**
1. 读取飞书待办表
2. 当日待办写入早报
3. 发送早报

---

## 七、防错机制

| 错误类型 | 判定 | 处理 |
|---------|------|------|
| 只写本地日志 | 执行层错误 | 触发 PUA 调试 |
| 创建定时提醒 | 违反用户指令 | 立即删除 |
| 找不到表格信息 | 信息缺失 | 立即询问用户，不假设，不猜测 |
| 事项名称含相对时间词 | 数据质量错误 | 自动剥离后再存储 |
