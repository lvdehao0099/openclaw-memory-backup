# 文件编辑安全规范

## 问题：edit 命令的精确匹配陷阱

**现象：**
```
Edit: in MEMORY.md failed
Could not be find the exact text in MEMORY.md
```

**原因：**
- `edit` 命令要求 `old_string` 必须**完全匹配**文件内容
- 包括所有空格、换行、标点
- 稍有差异就会失败

## 解决方案

### 方案A：精确匹配（仅用于极小修改）
- 先用 `read` 读取文件内容
- 复制要替换的**完整段落**（包括换行）
- 确保 `old_string` 和文件内容**一字不差**

### 方案B：覆盖写入（⭐ 推荐方案）
- 使用 `write` 命令**覆盖整个文件**
- 先读取原文件 → 修改内容 → 完整写回
- 避免部分匹配问题

### 方案C：append 追加（适用于新增）
- 使用 `write` 的 append 模式
- 在文件末尾添加新内容

## 安全操作流程（强制）

1. **读取** - 先用 `read` 确认文件当前内容
2. **复制** - 精确复制要替换的文本段落
3. **编辑** - 修改内容
4. **验证** - 编辑后再次读取确认成功

## 🚫 禁止操作（违者必究）

❌ **不要凭记忆写 `old_string`**
❌ **不要假设文件内容和上次一样**
❌ **不要在 `old_string` 里省略换行或空格**
❌ **不要在不确定时随意使用 edit**
