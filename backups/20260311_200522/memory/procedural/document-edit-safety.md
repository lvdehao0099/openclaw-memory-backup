# 文档编辑安全规范 - 【最高优先级】

## 🚨 铁律（不可违背）

### 1. 绝不覆盖原文
**除非用户明确说"覆盖"、"替换"、"删除原文"，否则：**
- ❌ 禁止使用 `write` 命令覆盖整个文档
- ❌ 禁止使用 `edit` 命令替换大段内容
- ✅ 必须使用 `append` 或 `insert` 追加新内容

### 2. 默认追加模式
**所有文档编辑默认使用追加/插入：**
- 新内容 → 添加到文档末尾
- 修改内容 → 在旧内容后添加"更新版"，并标注
- 对比内容 → 保留原文，在旁边添加新版本

### 3. 标记修改法
**如需修改现有内容，使用以下格式：**

```markdown
## 原文（保留）
[原内容保持不变]

---

## 更新版（新增）
[新内容]
**修改说明：** 原XX内容更新为XX，原因是XX

---

## 对比
| 原文 | 更新版 |
|------|--------|
| XX | XX |
```

### 4. 操作前确认
**修改文档前必须问自己：**
1. 用户是否要求覆盖原文？
2. 原文是否有重要信息需要保留？
3. 是否有更安全的追加方式？

**如果不确定 → 先问用户！**

## 📝 正确操作示例

### 示例1：添加新内容（推荐）
```
原文档：[原有内容]

操作：feishu_doc append
结果：[原有内容] + [新内容]
```

### 示例2：修改内容（标记法）
```
原文档：A

操作：不删除A，在后面添加A'
结果：
- A（原文，保留）
- A'（更新版，新增）
- 修改说明
```

### 示例3：版本对比
```
结果文档：
## 版本1（原文）
...

## 版本2（优化版）
...

## 修改对比
...
```

---

## 🚫 禁止操作（违者必究）

- ❌ 未确认就使用 `write` 覆盖文档
- ❌ 未备份就 `edit` 替换大段内容
- ❌ 删除用户的原文/配套文案/重要内容
- ❌ 以"优化"为名擅自修改用户原创内容

---

## ✅ 安全操作检查清单

修改文档前，必须完成以下检查：

- [ ] 是否确认用户要求覆盖原文？
- [ ] 是否使用了 append/insert 而非 write？
- [ ] 是否保留了原文的所有内容？
- [ ] 是否标注了修改说明？
- [ ] 用户是否能看到原文和修改版的对比？
