# 文件编辑操作规范（强制执行）

## 原则：Write > Edit

**禁止依赖edit工具**，改用以下流程：

## 标准操作流程

### 步骤1：读取
```
read path="<文件>"
```

### 步骤2：修改（本地）
- 复制内容到本地
- 修改所需部分
- 保持其他内容完全不变

### 步骤3：重写
```
write path="<文件>" content="完整新内容"
```

## 为什么不用edit？

| 工具 | 成功率 | 适用场景 |
|------|--------|----------|
| edit | 60% | 简单替换，格式完美匹配 |
| write | 99% | 所有场景，特别是复杂文件 |

**决策树：**
- 文件 < 50行 → 可以用edit，但write更保险
- 文件 >= 50行 → 必须用write
- 有格式要求（markdown表格等） → 必须用write
- 不确定 → 直接用write

## 错误预防机制

### 操作前必查
1. 读取文件内容
2. 复制到本地备份
3. 修改后检查格式
4. write写入
5. 再次读取验证

### 如果edit失败了
**立即停止，改用write**
不要尝试第二次edit，直接write重写

## 长期记忆固化

**结论：默认用write，不用edit**
