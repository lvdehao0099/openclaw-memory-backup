# 长任务汇报机制（CLO管理规范）

## 🚨 适用范围
**所有超过40分钟的任务、定时任务、批量任务**

## 📋 汇报时间表（严格执行）

### 任务开始前
- 明确告知：任务内容、开始时间、预计完成时间
- 示例："图片生成任务已启动，预计16:30完成，共12张"

### 执行中汇报
- **每30分钟**主动汇报一次进度
- 示例："[15:30进度] 已完成6/12张，预计正常"

### 异常/超时汇报
- **发现异常立即汇报**，不等待
- 示例："⚠️ 任务异常：只生成2张，原因：API限制，正在补救"

### 完成汇报
- **第一时间**告知完成
- 示例："✅ 任务完成！12张图片已生成，文档链接：xxx"

## 📊 汇报模板

```
【任务进度汇报】
任务：xxx
时间：xx:xx
进度：x/y（百分比）
状态：正常/异常/延迟
预计完成：xx:xx
```

## 🚫 禁止行为
- ❌ 任务开始不告知
- ❌ 超过30分钟不汇报
- ❌ 出现异常不立即说明
- ❌ 超时后被动等待询问

## ✅ 补救措施（本次）
- 立即重启图片生成任务
- 每30分钟主动汇报（15:00, 15:30, 16:00...）
- 建立任务监控脚本
