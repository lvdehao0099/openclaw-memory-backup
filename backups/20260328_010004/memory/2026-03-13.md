## 相对日期解析bug修复

**问题：** 录入时说"明天下午拜访"，事项名称记录了"明天"，但早报生成时已经是今天，"明天"变成错误时间

**根本原因：**
- 早报显示时直接使用事项名称原文，没有根据截止日期字段判断
- 事项名称中的相对日期（明天、后天）没有转换为绝对日期

**修复方案：**
1. 早报生成时必须使用「截止日期」字段判断时间
2. 根据截止日期计算距今天数：今天/明天/后天/M月D日
3. 事项名称中去掉相对日期描述

## 飞书表格 API v5.0 规范确定（22:00-23:30）

**问题：** 之前所有飞书表格创建方案都失败或有问题
- descendants 一次性注入：返回 success 但内容被忽略
- POST /children 追加：有空行（飞书自动生成空 text block）
- PATCH text：报 invalid param

**最终解决方案（v5.0）：**
三步走原子操作：
1. POST /children 创建空表容器（只传 block_type: 31 + row_size/column_size）
2. GET /children 获取幽灵块 ID（飞书自动生成的空 text block ID）
3. PATCH /blocks/{ghost_id} 使用 `update_text_elements` 覆盖更新

**关键发现：**
- 必须使用 `update_text_elements` 字段，不是 `text`
- 核心是"覆盖"不是"追加"
- 飞书创建空表时，会自动在每个单元格生成一个空的 text block（幽灵块）

**已创建规范文档：**
- ✅ `procedural/feishu_standard_v5.md` - v5.0 规范文档
- ✅ `scripts/feishu_table_manager.py` - FeishuTableManager 工具类

**工具类核心方法：**
- `create_table(doc_id, rows, cols)` - 创建空表容器
- `get_ghost_ids(doc_id, table_id)` - 提取幽灵块 ID
- `update_cell(doc_id, ghost_id, content, bold)` - 覆盖更新单个单元格
- `batch_update_content(doc_id, data, header_bold)` - 批量更新完整表格
- `create_table_with_data(title, data, folder_token)` - 一键创建带内容的表格

**测试结果：**
- ✅ 表格创建成功
- ✅ 零空行
- ✅ 表头加粗
