## 四、执行流程

### 4.1 新增待办流程

1. **用户提出待办**
   - 立即询问表格信息（如果未知）

2. **写入本地日志**
   - 路径：`memory/YYYY-MM-DD.md`

3. **写入飞书待办追踪表**
   - 使用飞书 API 写入
   - 日期字段使用毫秒时间戳

4. **确认双写成功**
   - 检查 API 返回 `code == 0`

### 4.2 更新待办流程

1. 查询飞书待办表获取 record_id
2. 更新状态字段
3. 同步更新本地日志

## 五、相对日期处理规则（重要！）

### 5.1 录入时转换

**把相对日期转换为绝对日期存储到「截止日期」字段：**

| 用户输入 | 存储值 |
|---------|--------|
| "明天下午" | 今天+1天的时间戳 |
| "下周三" | 下周三的时间戳 |
| "后天上午" | 今天+2天的时间戳 |

### 5.2 早报生成时判断

**使用「截止日期」字段判断时间，不依赖事项名称中的相对日期：**

| 截止日期 | 显示格式 |
|---------|---------|
| = 今天 | "今天 HH:MM" |
| = 明天 | "明天 HH:MM" |
| = 后天 | "后天 HH:MM" |
| 其他 | "M月D日 HH:MM" |

## 六、提醒方式（重要！）

| 方式 | 是否允许 | 说明 |
|------|:--------:|------|
| ❌ 创建定时提醒 | 禁止 | 用户不需要到时间提醒 |
| ✅ 在早报中提醒 | 允许 | 所有日程类待办都在当天早报里提醒 |

**早报流程：**
1. 读取飞书待办表
2. 当日待办写入早报
3. 发送早报

## 七、防错机制

| 错误类型 | 判定 | 处理 |
|---------|------|------|
| 只写本地日志 | 执行层错误 | 触发 PUA 调试 |
| 创建定时提醒 | 违反用户指令 | 立即删除 |
| 找不到表格信息 | 信息缺失 | 立即询问用户，不假设，不猜测 |
