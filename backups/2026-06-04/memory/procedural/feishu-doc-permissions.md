# 飞书文档创建规范 - 共同编辑版

## 核心原则
**使用用户 OAuth 身份创建文档**

当使用 `feishu_doc` 工具创建文档时，操作会以用户的身份执行：
- 文档创建者 = 用户本人
- 权限继承用户的默认权限
- 用户天然拥有编辑权限
- 无需额外配置权限

## 操作步骤

### 1. 创建文档
```
feishu_doc action="create" 
  folder_token="Thhrfa3QglB5amdIMpEcY2tAndd"  # 虾米🦞文件夹
  title="文档标题"
```

### 2. 关键要点
- **必须使用 folder_token 指定文件夹** → 确保文档在「虾米🦞」文件夹
- **创建后检查** → 确认用户可以编辑
- **无需额外权限设置** → OAuth 身份已确保权限

## 对比：之前的问题 vs 现在的方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 用户无法编辑 | 文档创建在默认位置（非虾米🦞文件夹） | 强制使用 folder_token |
| 权限问题 | 担心需要额外设置 | OAuth 身份自动解决 |

## 铁律
1. 每次创建飞书文档必须指定 folder_token
2. 创建后立即在对话中确认位置和权限
3. 绝不创建在默认/未知位置

## 生效时间
2026-03-12 起永久执行
