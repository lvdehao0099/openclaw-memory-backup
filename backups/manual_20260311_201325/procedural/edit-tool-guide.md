# Edit工具使用指南

## 为什么edit会失败？

edit工具要求 `old_string` 必须**完全匹配**文件中的内容，包括：
- 所有字符（包括空格）
- 换行符（\n vs \r\n）
- 行尾空格
- 缩进（tab vs 空格）

## 快速诊断步骤

### 方法1：使用诊断脚本
```bash
~/.openclaw/workspace/scripts/fix-edit-tool.sh <文件路径>
```

### 方法2：手动检查
```bash
# 1. 读取文件并显示不可见字符
cat -A <文件>

# 2. 转换换行符（Windows -> Unix）
sed -i 's/\r$//' <文件>

# 3. 删除行尾空格
sed -i 's/[[:space:]]*$//' <文件>
```

## 确保edit成功的最佳实践

### 步骤1：先读取文件
```
read path="<文件>"
```
复制输出的准确内容。

### 步骤2：使用write工具
对于复杂修改，直接用 write 重写整个文件，而不是 edit 修改部分。

### 步骤3：如果必须用edit
- 先用 `cat -A <文件>` 查看隐藏字符
- 确保 `old_string` 和文件内容完全一致
- 建议先用 `echo -n "<old_string>" | xxd` 检查hex

## 替代方案

**推荐：重要文件修改流程**
1. `read` 读取整个文件
2. 在本地修改内容
3. `write` 写入完整新内容

这样比 `edit` 更可靠，不容易出错。

## 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| Could not find the exact text | 文本不完全匹配 | 检查空格、换行符 |
| 修改后内容不对 | 匹配了错误的位置 | 使用更长的old_string确保唯一性 |
| 文件被其他进程修改 | 文件在外部被改动 | 重新读取文件再编辑 |
