# 公众号文章创作工作流程（龙虾团队版）

**创建时间：** 2026年3月11日  
**来源：** 依科公众号第1周文章创作经验总结  
**适用场景：** 高质量内容创作，需经得起推敲的正式文章

---

## 一、工作流程（必须严格执行）

### 阶段1：准备（30分钟）
1. **读取策略文档** - 必须完整阅读，确定主题方向
2. **阅读参考资料** - 历史讨论、案例素材
3. **明确约束条件** - 风格要求、字数、交付时间

**⚠️ 铁律：** 不读策略文档不动笔，主题必须严格符合策略要求

### 阶段2：团队讨论（1-2小时）
**参与角色：**
- 虾米（CLO）：统筹、决策、质量把控
- 文案虾：撰写、创意
- 老师傅虾：专业审核、事实核查

**讨论轮次（至少4轮）：**
1. **选题角度** - 切入点、受众代入感
2. **内容结构** - 大纲、核心论点、案例选择
3. **表达方式** - 语气、用词、专业度平衡
4. **结尾处理** - CTA、价值升华

**每轮输出：** 讨论记录、决策结论、待解决问题

### 阶段3：撰写初稿（2-3小时）
- 文案虾主笔
- 按讨论确定的结构执行
- 标注不确定的地方，留待审阅

### 阶段4：内部审阅（30分钟）
**老师傅虾审核清单：**
- [ ] 事实准确性
- [ ] 专业度是否恰当
- [ ] 有无过度承诺
- [ ] 是否踩竞品/贬低他人

**文案虾自检：**
- [ ] 逻辑流畅度
- [ ] 语言自然度
- [ ] 字数控制

**虾米把关：**
- [ ] 是否符合策略要求
- [ ] 风格是否一致
- [ ] 是否达标（不达标打回重写）

### 阶段5：路人审阅（关键！至少2轮）
**选择路人甲：**
- 目标受众代表（如维修工、设备部长）
- 非专业人士（普通读者视角）
- 批评型人格（容易挑刺的人）

**审阅维度：**
1. **可信度** - 数据是否真实、案例是否可信
2. **代入感** - 是否理解痛点、是否产生共鸣
3. **实用性** - 有无落地价值、行动指引是否清晰
4. **违和感** - 哪里读着不舒服、哪里像广告

**记录所有质疑，必须回应**

### 阶段6：修改完善（1-2小时）
- 整合路人意见
- 逐条修改或解释为什么不改
- 内部再审阅确认

### 阶段7：定稿交付
- 创建飞书文档（确认folder_token！）
- 完整记录：讨论过程 + 审阅意见 + 最终正文
- 交付前最后检查清单

## 二、常见错误（血泪教训）

### 错误1：主题偏离
**表现：** 凭记忆瞎编，不按策略要求写  
**后果：** 完全重写，浪费数小时  
**预防：** 动笔前抄写策略要求到文档顶部

### 错误2：缺少路人审阅
**表现：** 团队内部自嗨，自我感觉良好  
**后果：** 发布后被读者挑错，尴尬  
**预防：** 必须找2-3个路人甲，不同视角审阅

### 错误3：数据造假
**表现：** 编"85%概率""提升70%"等具体数字  
**后果：** 被专业人士识破，信任崩塌  
**预防：** 无真实来源就用模糊表述（"显著提升"）

### 错误4：贬低他人
**表现：** 为了突出AI，把人工经验说的一无是处  
**后果：** 得罪目标受众，温和科普变批判文  
**预防：** 强调"辅助"不是"替代"，尊重现有经验

### 错误5：文件放错位置
**表现：** 创建文档时没指定folder_token  
**后果：** 用户找不到，无法编辑  
**预防：** 创建前默念3次 folder_token

## 三、质量检查清单（交付前必查）

### 内容质量
- [ ] 主题严格符合策略要求
- [ ] 无编造数据
- [ ] 无贬低竞品/他人
- [ ] 语气温和、专业
- [ ] 案例真实可信

### 结构完整
- [ ] 开头有代入感
- [ ] 中间逻辑清晰
- [ ] 结尾有CTA
- [ ] 字数符合要求

### 流程完整
- [ ] 团队讨论记录完整
- [ ] 路人审阅记录完整
- [ ] 修改执行记录完整
- [ ] 文档在正确文件夹

## 四、核心原则

### 1. 慢就是快
- 严格流程看似慢，实则减少返工
- 1次做对 vs 3次返工，前者更快

### 2. 路人审阅不可替代
- 团队内部视角有盲区
- 外部视角能发现自嗨内容
- 被路人挑错，好过被客户挑错

### 3. 质量不达标不交付
- 宁可延期，不交垃圾
- 1篇精品 > 3篇平庸

### 4. 记录完整过程
- 讨论过程、审阅意见、修改记录都要存
- 便于复盘、迭代、证明工作量

## 五、时间预估（参考）

| 阶段 | 时间 | 累计 |
|------|------|------|
| 准备 | 30min | 30min |
| 团队讨论 | 1-2h | 2-3h |
| 撰写初稿 | 2-3h | 4-6h |
| 内部审阅 | 30min | 4.5-6.5h |
| 路人审阅 | 1h（等待） | 5.5-7.5h |
| 修改完善 | 1-2h | 6.5-9.5h |
| 定稿交付 | 30min | 7-10h |

**说明：** 本次实际耗时约6小时（含前期绕圈子），合理范围。

## 七、使用说明

**启动任务时：**
1. 读取本文档
2. 确认本次任务适用此流程
3. 严格执行，不走捷径

**遇到冲突时：**
- 想快 vs 按流程 → 选择按流程
- 想省路人审阅 → 必须执行
- 数据不确定 → 模糊处理，不编造

**复盘时：**
- 对照本文档检查执行是否到位
- 记录新教训，更新本文档
