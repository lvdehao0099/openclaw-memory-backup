2. **提取待办** - 从历史记录中提取真实的待办事项
3. **生成内容** - 按格式生成早报
4. **发送** - 7:50准时发送

## 相对日期解析规则（重要！）

**问题：** 用户录入时说"明天下午拜访"，事项名称里记录了"明天"，但早报生成时已经是今天，"明天"就变成了错误的时间。

**解决方案：**
1. **早报生成时必须使用截止日期字段判断**，不依赖事项名称中的相对日期
2. **显示格式：** 根据截止日期计算距今天数
   - 截止日期 = 今天 → "今天 HH:MM"
   - 截止日期 = 明天 → "明天 HH:MM"
   - 截止日期 = 后天 → "后天 HH:MM"
   - 其他 → "M月D日 HH:MM"
3. **事项名称显示：** 去掉事项名称中的相对日期描述（明天、后天、下周X等）

**示例：**
```
原事项名称: 益大橡胶客户拜访 - 明天下午14:00
截止日期: 2026-03-15 14:00
今天是: 2026-03-14

正确显示: 益大橡胶客户拜访（明天 14:00）
错误显示: 益大橡胶客户拜访 - 明天下午14:00（今天看就是错的！）
```

## 铁律
- ✅ 生成早报前**必须**读取飞书待办表
- ✅ 戒烟天数必须准确（从quitting-smoking.md读取）
- ✅ **时间判断必须用截止日期字段，不用事项名称中的相对日期**
- ❌ **禁止**虚构待办事项
- ❌ **禁止**询问吕总已经记录在表中的信息
- ⚠️ 不确定的数据标注"待确认"
