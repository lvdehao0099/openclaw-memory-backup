# 微信热点采集系统配置
# 采集微信公众号/搜一搜热榜前10名，14天滚动保留

## 系统架构

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  微信热榜采集    │────▶│  飞书Bitable     │────▶│  14天自动清理    │
│  (每日3次)      │     │  热点雷达追踪表   │     │  (过期自动删)   │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

## 采集频率

- **每日3次**：08:00 / 14:00 / 20:00
- **每次采集**：微信热榜前10名
- **保留策略**：14天后自动删除

## 飞书Bitable字段（完整版）

| 字段 | 类型 | 说明 |
|------|------|------|
| 日期 | 日期 | 采集时间 |
| 热点标题 | 文本 | 热点内容 |
| 热度排名 | 数字 | 1-10 |
| 来源 | 单选 | 微信搜一搜/公众号/其他 |
| 原文链接 | URL | 原始链接 |
| 相关度 | 单选 | 高/中/低 |
| 选题建议 | 文本 | AI生成的选题角度 |
| 状态 | 单选 | 待讨论/已选题/已发布/已放弃 |
| 过期时间 | 日期 | 14天后自动过期 |

## 数据源配置

### 主数据源（优先级排序）

1. **搜狗微信搜索热榜**
   - URL: https://weixin.sogou.com/
   - 更新频率：实时
   - 采集方式：网页爬虫

2. **新榜（微信榜单）**
   - URL: https://www.newrank.cn/
   - 更新频率：每小时
   - 需要：API Key（可选）

3. **清博大数据**
   - URL: https://www.gsdata.cn/
   - 更新频率：每小时
   - 需要：API Key（可选）

4. **微信搜一搜热词**
   - 来源：微信指数
   - 更新频率：每日
   - 采集方式：手动整理（备用）

## 自动清理机制

### 清理策略
- **触发时间**：每日00:00
- **清理条件**：过期时间 < 当前时间
- **清理动作**：自动删除过期记录
- **备份**：删除前自动备份到 GitHub

### 保留价值判断
| 时间 | 价值 | 动作 |
|------|------|------|
| 0-3天 | ⭐⭐⭐⭐⭐ 最高 | 优先选题 |
| 4-7天 | ⭐⭐⭐⭐ 高 | 可用 |
| 8-14天 | ⭐⭐⭐ 中 | 参考 |
| >14天 | ⭐⭐ 低 | 自动删除 |

## 选题工作流

### 每日早上（8:30）
1. 查看当日采集的微信热榜前10
2. 筛选高/中相关度热点
3. 结合AI老师傅业务生成选题
4. 将选中热点状态改为"已选题"

### 每周一
1. 回顾本周所有热点
2. 分析哪些热点适合深度文章
3. 规划下周选题计划

## 选题优先级算法

### 评分维度
| 维度 | 权重 | 说明 |
|------|------|------|
| 热度排名 | 30% | 排名越靠前越好 |
| 相关度 | 40% | 高相关>中相关>低相关 |
| 时效性 | 20% | 3天内>7天内>14天内 |
| 可操作性 | 10% | 容易切入>需要包装 |

### 推荐公式
```
选题价值 = 热度排名×0.3 + 相关度×0.4 + 时效性×0.2 + 可操作性×0.1
```

1. 手动整理微信热榜前10
2. 使用「微信文章搜索」技能补充
3. 关注行业公众号，人工筛选
