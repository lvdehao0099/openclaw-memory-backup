# 微信热点采集任务操作规范

## 任务说明
自动采集微信热榜前10名，保存到飞书Bitable热点雷达追踪表

## 定时任务配置

| 班次 | 时间 | Cron表达式 |
|------|------|-----------|
| 早班 | 08:00 | `0 8 * * *` |
| 午班 | 14:00 | `0 14 * * *` |
| 晚班 | 20:00 | `0 20 * * *` |

## 执行流程

### 1. 采集数据
- 采集微信搜一搜热榜前10名
- 获取热点标题、来源、原文链接、热度排名

### 2. 计算相关度
使用AI分析热点与AI老师傅业务的相关度（0-100分）

### 3. 生成选题建议
根据相关度生成选题建议：
- 相关度 ≥70分：🔥 高相关，建议深度跟踪
- 相关度 40-69分：📊 中度相关，参考使用  
- 相关度 <40分：📌 低相关，暂不跟进

### 4. 写入飞书Bitable

**表格地址：** https://ai-zhineng.feishu.cn/base/Xb0abYaXlayTnNsU2Oocrp42nUh

**字段映射：**
| 字段 | 说明 |
|------|------|
| 热点标题 | 热点标题（主字段） |
| 来源 | 热榜来源 |
| 原文链接 | 热点文章URL |
| 热度排名 | 1-10排名 |
| 相关度 | AI计算的相关度分数 |
| 选题建议 | AI生成的选题建议 |
| 状态 | 待处理/处理中/已发布/已过期 |
| 过期时间 | 自动计算（创建时间+14天） |

### 6. 发送简报
生成采集简报发送到飞书，包含：
- 本次采集数量
- 高相关热点列表
- 选题建议汇总
- 表格链接

## 数据保留策略

- **保留期：** 14天
- **自动清理：** 每天00:00清理过期记录
- **备份：** 重要热点手动标记为"长期保留"

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 采集失败 | 重试3次，失败则发送告警 |
| 表格写入失败 | 检查字段名，确认API权限 |
| 空列未删除 | 立即清理，更新规范 |
