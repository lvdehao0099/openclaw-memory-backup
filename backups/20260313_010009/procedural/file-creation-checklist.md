# 文件创建规范检查清单
# 每次创建文件前必须执行

## 前置检查（必须回答）

### 1. 这是什么类型的文件？
- [ ] 日记/日志 → 放入 `memory/YYYY-MM-DD.md`
- [ ] 工作成果/报告 → 放入 `memory/` 或 `research/`
- [ ] 流程规范 → 放入 `procedural/`
- [ ] 知识记录 → 放入 `semantic/`
- [ ] 临时文件 → 放入 `/tmp/` 或 `research/`
- [ ] 飞书协作文档 → **直接在飞书创建，不要本地写**

### 2. 如果是飞书相关文档：
- [ ] 是否已确认飞书目标文件夹？（必须是「虾米🦞」文件夹）
- [ ] 是否使用 feishu_doc 工具直接创建？
- [ ] 本地草稿是否放在 `memory/` 下而非根目录？

### 3. 路径确认
创建前必须明确说出完整路径：
```
我将创建文件：/root/.openclaw/workspace/{正确目录}/文件名
```

## 违规后果
- 根目录出现 .md 文件 = 立即触发错误
- 必须立即移动到正确位置
- 记录到 BUG修复记录.md

## 常见错误模式（我必须警惕）
1. ❌ 直接写 `write file_path="文件名.md"` → 默认在当前目录
2. ❌ 忘记指定完整路径
3. ❌ 把飞书文档先在本地写，忘记上传

## 正确示例
✅ `write file_path="/root/.openclaw/workspace/memory/2026-03-11-热点日报.md"`
✅ `feishu_doc create` 直接在飞书创建
